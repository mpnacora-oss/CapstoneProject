# PC-ALLEY ERP: Software Requirements

Based on the provided design reference and the actual technical implementation of the PC-ALLEY system, here is the updated Software Requirements table for your documentation.

### Table 3
**Software Requirements**

| Software | Description |
| :--- | :--- |
| **Visual Studio Code** | The developers use Visual Studio Code as the primary code editor for building, debugging, and managing the PC-ALLEY system. |
| **Web Browser** | Users can access the **PC-ALLEY ERP: Centralized Sales and Management System** through modern web browsers (e.g., Chrome, Firefox, Edge) using a secure internet connection. |
| **Hosting Service** | An online hosting platform is used to deploy the system, store application data, and manage the centralized database for all modules. |
| **Node.js** | Node.js serves as the server-side runtime environment, handling system logic, API processing, and secure database interactions. |
| **Next.js & React** | These frameworks are utilized to build a responsive, dynamic, and high-performance user interface for the ERP system. |
| **JavaScript** | JavaScript is the primary programming language used to add interactivity, handle front-end state management, and implement complex business logic. |
| **Tailwind CSS** | A modern, utility-first CSS framework used to design a professional, responsive, and visually consistent interface across all devices. |
| **GitHub** | GitHub is used for version control, collaboration, and source code management among the development team. |
| **MySQL** | MySQL is used as the database management system to store and organize sales, inventory, and user data in a centralized relational database. |
